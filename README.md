# منظومة المحاسبة الاحترافية (بايثون)

هذه المنظومة مصممة لتلبية احتياجات المحاسبين في إدارة الدورة المحاسبية بشكل كامل واحترافي.

## المميزات الرئيسية

1. **دليل حسابات مرن**: يدعم المستويات، التصنيفات، والترقيم التلقائي الذكي.
2. **قيود يومية ذكية**:
   - التحقق الفوري من توازن القيد (مدين = دائن).
   - تنبيه باللون الأحمر في حال عدم التوازن.
   - دعم مراكز التكلفة لكل سطر في القيد.
3. **دفتر الأستاذ العام**: عرض تفصيلي لحركة أي حساب مع احتساب الرصيد التراكمي آلياً.
4. **ميزان المراجعة**: تقرير شامل يوضح أرصدة جميع الحسابات مع التحقق من توازن الميزان الإجمالي.
5. **واجهة مستخدم احترافية**: تدعم اللغة العربية (من اليمين إلى اليسار) وتصميم عصري.

## متطلبات التشغيل

- Python 3.x
- المكتبات: `PyQt6`, `SQLAlchemy`, `pandas`

## التثبيت عبر بيئة افتراضية

1. أنشئ البيئة الافتراضية:

   ```bash
   python -m venv .venv
   ```

2. ثبّت الاعتمادات:

   ```bash
   .venv\Scripts\python -m pip install -r requirements.txt
   ```

## طريقة التشغيل

1. تأكد من تثبيت المكتبات المطلوبة:

   ```bash
   .venv\Scripts\python -m pip install -r requirements.txt
   ```

2. قم بتشغيل الملف الرئيسي:

   ```bash
   .venv\Scripts\python main.py
   ```

## تشغيل واجهة Flask الجديدة

1. ثبّت الاعتمادات المحدثة:

   ```bash
   .venv\Scripts\python -m pip install -r requirements-web.txt
   ```

2. شغّل خادم الويب:

   ```bash
   .venv\Scripts\python run_web.py
   ```

3. افتح المتصفح على:

   ```text
   http://127.0.0.1:5001
   ```

المستخدم الافتراضي في أول تشغيل:

```text
admin@example.com / Admin@12345
```

مستخدم تجريبي إضافي لاختبار صلاحيات المشاريع:

```text
field.manager@example.com / Manager@12345
```

المشاريع التجريبية التي يتم توليدها تلقائياً:

- `MAIN`: مشروع جاهز مع سنة مالية، حسابات، ميزانية، وقيود يومية.
- `FIELD-SVC`: مشروع جاهز ثانٍ لاختبار العمل بين أكثر من مشروع.
- `BUILD-OPS`: مشروع غير مكتمل الإعداد لاختبار بوابة الجاهزية.

المستخدم `field.manager@example.com` مرتبط فقط بمشروع `FIELD-SVC` لإظهار اختلاف محفظة المشاريع من مستخدم لآخر.

ضمن النموذج الأولي الحالي ستجد أيضاً:

- شاشة مستقلة لتسجيل الدخول.
- شاشة تسجيل حساب جديد.
- شاشة Onboarding لإضافة الشركة ثم إنشاء أول مشروع.
- شاشة مستقلة لاختيار المشروع.
- شاشة إعداد للمشروع غير الجاهز.
- مساحة عمل مالية منفصلة لكل مشروع ولكل سنة مالية.
- تبويب لتحويلات المشاريع بين مشروع جاهز وآخر.
- تحويلات المشاريع تنشئ قيوداً محاسبية مقابلة في المشروع المصدر والمشروع الوجهة وتظهر في الأستاذ وميزان المراجعة.
- تصدير Excel وPDF على مستوى المشروع أو السنة المالية.

## العزل حسب الشركة وبداية RBAC/ABAC

- كل جلسة تعمل على `شركة نشطة` (Active Company).
- المشاريع والبيانات المعروضة للمستخدم تكون ضمن الشركة النشطة فقط.
- إنشاء المستخدمين وإدارتهم يتطلب صلاحية على مستوى الشركة (`company.users.manage`).
- إنشاء المشاريع وإعداداتها (سنوات مالية/حسابات/ميزانيات) يتطلب صلاحية (`project.manage`).
- إدخال القيود والتحويلات يتطلب صلاحية (`project.finance.write`) مع تحقق إضافي من حالة السنة المالية (ABAC أولي).

> الأدوار الحالية كبداية: `company_owner`, `finance_manager`, `accountant`, `employee`, `viewer`.

## التصدير لويندوز 10 و11

يمكنك إنشاء نسخة قابلة للتشغيل على Windows 10 وWindows 11 باستخدام PyInstaller:

```powershell
.\build_windows.ps1
```

الوضع الافتراضي الآن يبني نسخة الويب (Flask API + Web UI).

لإنشاء نسخة سطح المكتب القديمة (PyQt) بشكل صريح:

```powershell
.\build_windows.ps1 -Target desktop
```

لإنشاء ملف تنفيذي واحد فقط (`.exe`) بدل مجلد كامل:

```powershell
.\build_windows.ps1 -OneFile
```

سيتم إنشاء النسخة داخل:

```text
dist\AccountingSystem
```

أو في وضع الملف الواحد:

```text
dist\AccountingSystem.exe
```

ولتوليد ملف واحد لنسخة سطح المكتب القديمة:

```powershell
.\build_windows.ps1 -Target desktop -OneFile
```

وفي حالة بناء النسخة المجلدية (الافتراضي) سيتم أيضاً إنشاء ملف مضغوط جاهز للنشر:

```text
dist\AccountingSystem-windows.zip
```

ملاحظة: يفضل بناء النسخة على نفس نوع النظام المستهدف (64-bit) واختبارها على Windows 10 وWindows 11 قبل التوزيع النهائي.

## تجهيز المشروع للنشر على GitHub

1. تهيئة Git لأول مرة داخل المجلد:

   ```powershell
   git init
   git add .
   git commit -m "Initial commit"
   ```

2. أنشئ مستودعاً جديداً على GitHub (فارغاً بدون ملفات).

3. اربط المشروع المحلي بالمستودع ثم ارفع الفرع الرئيسي:

   ```powershell
   git branch -M main
   git remote add origin <YOUR_GITHUB_REPO_URL>
   git push -u origin main
   ```

4. بعد الرفع، GitHub Actions سيبني نسخة Windows تلقائياً عبر الملف:

   ```text
   .github/workflows/build-windows-exe.yml
   ```

5. ستجد المخرجات في صفحة Action Artifacts باسم:

   ```text
   accounting-system-windows-exe
   ```

## أندرويد

النسخة الحالية مبنية بواجهة `PyQt6 Widgets` لسطح المكتب، ولذلك لا توجد لها حالياً عملية تصدير مباشرة ومدعومة إلى Android ضمن هذا المشروع. تشغيلها على Android يتطلب مساراً منفصلاً، مثل:

- إعادة بناء الواجهة بتقنية تدعم Android بشكل مباشر.
- أو تحويل النظام إلى تطبيق ويب أو واجهة جوال مع إعادة استخدام منطق الأعمال.

بمعنى عملي: Windows 10 و11 جاهزان للتجهيز والتصدير الآن، أما Android فيحتاج مرحلة تطوير منفصلة وليس مجرد إعداد venv.

## هيكلية المشروع

- `main.py`: الواجهة الرئيسية والربط بين الصفحات.
- `models.py`: تعريف قاعدة البيانات (SQLite).
- `chart_of_accounts.py`: إدارة دليل الحسابات.
- `journal_entries.py`: إدارة القيود اليومية.
- `reports.py`: دفتر الأستاذ وميزان المراجعة.
